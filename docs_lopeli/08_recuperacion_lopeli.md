# 08_recuperacion_lopeli

## Recuperación ante desastres

La recuperación busca volver a operar con el menor tiempo y pérdida posible después de un incidente grave.

### Mejora tecnológica

- Replicación de base de datos para alta disponibilidad.
- Backups automáticos con restauración verificada.
- Separación de servicios críticos por capas.
- Registro centralizado para reconstruir la línea de tiempo del incidente.

### Plan de continuidad

1. Contener el incidente y aislar el sistema afectado.
2. Preservar evidencias y registros.
3. Restaurar desde respaldo confiable.
4. Validar integridad de datos y funcionalidades.
5. Reanudar operación controlada.

### Objetivos

- RTO: hasta 2 horas para el portal crítico.
- RPO: hasta 15 minutos para datos transaccionales.

### Conclusión

La recuperación no debe improvisarse. Para una fintech, la restauración rápida y validada es parte del control de riesgo y de la confianza del negocio.
